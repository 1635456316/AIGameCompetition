# 关卡编辑器

可视化编辑 AIGameCompetition 关卡布局的纯前端工具，与游戏共用 `assets/levels/*.json` 数据格式。

## 打开方式

**必须通过本地 HTTP 服务访问**（浏览器 `fetch` 无法读取本地 file:// 路径）。

在项目根目录执行：

```bash
npx serve
```

然后访问：

```
http://localhost:3000/ExtraTools/关卡编辑器/
```

## 界面说明

| 区域 | 功能 |
|------|------|
| **顶栏** | 新建 / 打开 / 保存 JSON、撤销重做、关卡切换、选择/平移工具、网格吸附、缩放 |
| **左栏 · 素材库** | 拖拽或点击选中后点击地图，放置平台、竖墙、可破坏墙、回血道具、机关、敌人、出生点/Boss 标记 |
| **中间 · 视口** | 720px 高关卡画布；Ctrl+滚轮缩放；平移工具或中键拖动画布 |
| **右栏 · 属性** | 选中元素的精确坐标与参数 |
| **右栏 · 层级** | 全部对象列表，点击选中 |
| **右栏 · 关卡** | 关卡宽度、标题、背景图/BGM/PV URL、Boss 触发距离等 |

## 保存到游戏

编辑器使用浏览器的 **File System Access API** 直接写入本地文件：

1. **首次保存**某关卡时，会弹出文件选择框，请选中 `assets/levels/level_N.json`（例如 `level_1.json`）
2. **之后点击「保存」或按 Ctrl+S**，会直接覆盖该文件，不再重复下载
3. 文件关联会按关卡 ID 记住（存在浏览器 IndexedDB 中），切换关卡后会自动恢复对应关联
4. 若浏览器不支持（如 Firefox），会回退为下载 JSON，需手动替换到 `assets/levels/`

新建关卡首次保存后，还需在 `assets/levels/manifest.json` 的 `levels` 数组中追加路径。

保存后刷新游戏页面即可生效。

## 关卡背景图

`bgUrl` 指向关卡横向背景图（如 `assets/UI/第一关背景图.png`）。游戏中按高度等比缩放并水平平铺；编辑器画布会同步预览。留空则使用程序生成的渐变/视差背景。

## 坐标约定

- 世界高度固定 **720px**，地面顶边 Y = **656**
- 平台：`[x, y, count]`，x/y 为第一块中心，每段宽 **96px**
- 竖墙 / 机关矩形：x/y 为矩形中心
- 玩家出生：`playerStart.x` + `yOffset`（距底边像素）

## 竖墙

`walls` 数组中的竖墙会在游戏中加入 `groundSolids` 碰撞组，**L 键冲刺无法穿过**。

## 可破坏墙

`destructibleWalls` 与竖墙格式相同（`x/y/w/h`），额外字段 `hp`（默认 3）表示受击次数。玩家普攻、剑气、第三段冲刺均可扣耐久，归零后墙体消失。

## 回血道具

`pickups` 数组，当前支持 `{ "type": "health", "x", "y", "amount": 30 }`。玩家触碰且未满血时拾取回血。

## 快捷键

| 键 | 功能 |
|----|------|
| V | 选择工具 |
| H | 平移工具 |
| Delete | 删除选中元素 |
| Ctrl+Z / Ctrl+Y | 撤销 / 重做 |
| Ctrl+S | 保存 |
