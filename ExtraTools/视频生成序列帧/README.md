# 视频生成序列帧

一个纯前端的本地工具，把视频中手动挑选的若干帧拼接成 sprite sheet（序列帧图），并导出 PNG + JSON 元数据，方便接入 Phaser/Cocos 等游戏引擎。

- 纯浏览器运行，零上传，零依赖（不需要 Node / npm / 服务器）
- 双击 `index.html` 即可使用
- 输出 `.png` 序列帧图 + 同名 `.json`（含每帧时间戳与位置）

## 打开方式

1. 用 Chrome / Edge / Firefox 等现代浏览器打开本目录下的 `index.html`
   - 推荐 Chrome / Edge，支持 `requestVideoFrameCallback`，抓帧更稳定
2. 如果直接双击打不开（被系统当作其它程序处理），可以右键 → 选择浏览器打开
3. 全部处理在本地完成，**视频文件不会被上传**

## 操作流程

1. 顶部「选择视频文件」按钮，或者把视频文件直接拖到中间的播放器区域
2. 用进度条 / 播放暂停 / 上一帧下一帧定位到想要的画面
3. 点击「★ 标记当前帧」按钮，或者按快捷键 `M`，把当前帧加入"已标记列表"
4. 标记的帧会按时间自动排序，并显示为带时间戳的缩略图
   - 点击缩略图 = 跳转到该时间
   - 鼠标悬停时右上角 `×` = 删除该标记
5. 在「导出配置」面板里调整参数（见下文）
6. 点击「生成预览」，等待进度条走完，下方会显示拼接好的预览图
7. 点击「导出 PNG + JSON」，浏览器会自动下载两个文件

## 动画预览

「已标记帧」上方提供一个动画预览面板，可以直接把已标记的帧按指定间隔顺序播放出来，效果就是导出后的序列帧动画效果，方便在导出前调整时间点和节奏：

- **▶ 播放动画 / ■ 停止**：开始 / 停止预览
- **间隔 (ms) ↔ FPS**：两个输入框双向联动，改一个另一个自动更新。默认 100ms（10 FPS）
- **循环**：勾选后到末帧自动回到首帧（默认开）
- **来回播放**：勾选后按 1→N→1 来回播放（pingpong 模式）
- **第 x / N 帧**：当前播放到第几帧
- 增删标记、加载新视频、清空标记时会自动停止预览
- 预览取的是「标记当时主视频画面」的快照（最大 640×360），无需再次 seek 视频，因此切换流畅

## 背景抠图（Chroma Key）

视频帧本身是**不透明**的（白色背景 = 白色像素，并非透明），仅勾选导出配置里的"透明背景"是不够的——那个开关只控制各帧之间的"空白区域"，无法处理帧画面里的底色。要让导出的序列帧带真透明，需要用「背景抠图」面板：

- **启用抠图**：勾选后下方所有控件生效；缩略图、动画预览、最终导出都会应用同样的抠图设定
- **抠除颜色**：要被抠成透明的颜色。默认 `#ffffff` 白色
- **从画面取色**：最简单的用法。点击此按钮后再去点击 **视频画面 / 动画预览画布** 上的背景区域，会自动把那个像素的颜色填进去并启用抠图。按 `Esc` 可取消取色模式
- **容差**：颜色匹配的允许范围（0~200，欧几里得距离）。值越大，越多接近的颜色会被一起抠掉
- **边缘羽化**：在"完全抠掉"和"完全保留"之间做平滑过渡（0~100）。值大些可缓解锯齿/光晕，但会让边缘半透明
- **仅从边缘抠除（保留内部）**：默认勾选。**从画布四条边缘出发做 BFS 泛洪填充**，只把"和边缘连通"的同色像素抠掉。角色身上的同色高光（比如眼白、身体的白色反光）因为被周围非同色像素围着，会被完整保留，不会被误伤。取消勾选会变回"全图所有同色像素都抠掉"的传统模式
- **去边 (spill)**：移除半透明边缘里残留的键色（例如绿幕里的绿边、白底里的白边），可减轻"发白"现象

效果在标记后即时可见——缩略图与动画预览都会显示抠图后的结果（透明像素显示为棋盘格背景），可以一边调参数一边肉眼对比。最终导出的 PNG 会包含完整的 alpha 通道，在 Phaser / Photoshop 中打开就是真透明。

> 提示：白底动画字符通常 `容差=30~60、羽化=10~20、勾选"仅从边缘抠除"` 就能得到干净结果，且角色身上的眼白/高光都能完整保留。若角色的某些白色部分意外没被保留（说明它"漏"到画布边缘了），可以稍微降低容差再试。

## 一键均匀标记

如果你只是想从视频里均匀抽取 N 帧做序列帧（最常见的场景），不用手动一帧一帧地标记：

1. 加载视频后，控制行里有「平均 [N] 帧 / ⚡ 一键均匀标记」
2. 填一个目标帧数（默认 12），点按钮
3. 工具**模拟人手动拖动进度条**：用 [`HTMLMediaElement.fastSeek()`](https://developer.mozilla.org/zh-CN/docs/Web/API/HTMLMediaElement/fastSeek) 跳到最近关键帧 + `requestVideoFrameCallback` 等下一帧呈现 + 直接抓快照
4. 每帧只需 ~10-50ms，10 帧不到半秒就能跑完
5. 结束后已有的所有标记功能（动画预览、抠图、导出）立刻可用

> 注意：`fastSeek` 为了快速会**跳到目标时间附近的"最近关键帧"**，所以实际标记的时间点可能和你输入的目标点偏几十毫秒。`mark.time` 记录的是 `video.currentTime`（实际帧时间），所以进度条 tick 和动画预览都是准确的。如果对时间点精度要求很高，建议手动 `★ 标记当前帧`。

## 进度条标记

时间轴上每个已标记的帧位置都会显示一个**橙黄色 tick**（带向下的小三角），方便你一眼看清楚标记是否覆盖均匀。点击 tick 即可把视频跳转到那一帧的位置。

## 快捷键

| 按键 | 功能 |
| ---- | ---- |
| `Space` | 播放 / 暂停 |
| `←` / `→` | 上一帧 / 下一帧（按当前 FPS 计算步长） |
| `M` | 标记当前帧 |

## 导出配置说明

| 配置项 | 说明 |
| ---- | ---- |
| 列数 N | sprite sheet 每行帧数。`N=1` 单列竖排，`N=帧数` 单行横排，其它为网格 |
| 使用原始分辨率 | 勾选时直接用视频原始 `videoWidth × videoHeight` 作为每帧尺寸 |
| 目标宽 / 高 | 取消上面勾选后生效，可指定每帧的目标像素尺寸 |
| 缩放模式 | 自定义尺寸时的适配方式：`contain`（等比留空）/ `cover`（等比裁切）/ `stretch`（拉伸） |
| 帧间距 | 各帧之间的像素间隙（含外边距） |
| 透明背景 | 勾选时背景透明（PNG 透明像素） |
| 背景颜色 | 取消透明后生效 |

## 输出文件

- `sprites_<原文件名>_<时间戳>.png` —— 拼接好的 sprite sheet
- `sprites_<原文件名>_<时间戳>.json` —— 元数据，结构如下：

```json
{
  "source": "原视频文件名.mp4",
  "videoSize": { "width": 1920, "height": 1080 },
  "frameWidth": 256,
  "frameHeight": 256,
  "columns": 4,
  "rows": 3,
  "padding": 0,
  "sheetWidth": 1024,
  "sheetHeight": 768,
  "transparent": true,
  "background": null,
  "fit": "origin",
  "chromaKey": {
    "color": "#ffffff",
    "tolerance": 30,
    "feather": 15,
    "spill": false,
    "flood": true
  },
  "count": 12,
  "frames": [
    { "index": 0, "time": 1.234, "x": 0,   "y": 0, "w": 256, "h": 256 },
    { "index": 1, "time": 2.456, "x": 256, "y": 0, "w": 256, "h": 256 }
  ]
}
```

## 性能说明

- 标记一帧时，工具会**当场**从主视频画面抓一张快照（最大 1920×1080），存到内存里
- 后续的缩略图、动画预览、抠图、最终导出**都基于这张快照**
- 因此 **"生成预览"和"导出 PNG"几乎是瞬间完成的**，不会再去重新 seek 视频
- 如果导出尺寸大于快照（比如视频是 4K 但只抓到 1920），导出时会做一次轻微的上采样
- 想换更高分辨率的快照？刷新页面重新标记即可（上限固定 1920×1080，避免 4K 视频炸内存）

## 常见问题

- **导出的某些帧看起来"差一帧"？**
  浏览器对任意时间 seek 的精度受视频关键帧分布影响。可以适当微调标记的时间，或使用 `←` / `→` 按 FPS 步进精确定位后再标记。
- **拼出来的图太大失败？**
  浏览器 Canvas 单边一般最大 16384px，超出会失败。可减少列数或勾选自定义尺寸并缩小每帧尺寸。
- **支持哪些视频格式？**
  取决于浏览器解码能力。常见的 `mp4 (H.264)` / `webm` / `mov (H.264)` 都可以，比较新的 H.265/HEVC 可能不支持。
- **能不能不导出 JSON？**
  目前一次性导出 PNG 和 JSON 两个文件。JSON 留着方便接 Phaser 的 `load.spritesheet` / `load.atlas`，不用可直接删除。
